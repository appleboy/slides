## 議程簡介

本場演講探討真實挑戰：AI Agent 的大規模導入，使專案數量在短期內從 300 暴增至約 1,500，DevOps 壓力呈指數而非線性成長。

## 三大核心策略

### 1. AI 驅動的 PR 品質關卡

- 跨所有專案自動化進行架構審查與安全審查
- 在規模化情境下維持程式碼品質與安全性

### 2. 加速團隊 Onboarding

- 利用 AI 輔助將新團隊整合時間從約 3 個月壓縮至約 2 週

### 3. MCP 服務的 GitOps Pipeline

- 使用 Gitea、API 與 Gitea Actions
- 單一 `git push` 觸發完整自動化部署流程

## 聽眾收穫

- 將 AI 應用於 PR review，在規模化下維持品質與安全
- 大幅壓縮團隊 onboarding 時間軸
- 實作 GitOps 驅動的自動化服務部署 Pipeline
- 經過 300 → 1,500 專案成長曲線驗證的 DevOps 擴展策略
